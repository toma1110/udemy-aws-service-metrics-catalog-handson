# AWSサービス別メトリクスカタログ ハンズオン資料

## 目的
- 本講座の「何を最初に見るか」を体系化した実践チェックリストを提供する。
- 各レクチャーで登場するサービスを、**アラーム / ダッシュボード / 調査 / イベント・ログ・監査** の観点で即座に見分ける習慣を作る。
- 学習者が実務で同じ判断を再現できることを目標とする。

## 前提
- AWSアカウントは**必須ではない**（この教材は主に思考トレーニング向け）。
- 受講者がサービス名と主要メトリクスの意味をある程度理解していること。
- 可能なら AWS コンソールや `CloudWatch`、`EventBridge` の基本画面に事前に慣れていること（任意）。

## 準備
1. `handson/service_metric_catalog.md` を開く。
2. 受講しているセクションの講義名を確認し、該当サービスを抽出する。
3. `主要カタログ` の `最初に見る指標 / シグナル` と `主な用途` を照らし合わせる。

## 演習手順
1. 各レクチャーごとに、表示したい監視領域を1つ選ぶ。
2. 対応サービスの行を見つけ、優先して確認すべきメトリクスを3〜5個選ぶ。
3. メトリクスの変動と、講義で触れたアラート条件・イベント・ログとの因果を文章化する。
4. 該当領域の初動対応を、下記テンプレートで1〜2行で記録する。
   - 何が起きているか
   - どの指標が先行したか
   - まず何を見直すか（ダッシュボード / アラーム / イベント）
   - 次の切り分けステップ

## 期待結果
- 受講者がサービス別に「最初に見るべき指標」を1画面で選び、調査順序を説明できる。
- 講義ごとのテーマと整合した運用判断シーケンスを作成できる。

## コスト・安全
- このハンズオンは文書ベースで、AWSリソース作成・削除は行わない。
- 受講者が実検証する場合は、**本番環境の重要なアラーム値変更は行わない**ことを明記する。

## クリーンアップ
- 本資料はGit管理対象の静的Markdownのみ。
- 実行後に削除すべきアーティファクトはない。

## トラブルシュート
- サービス行が見つからない場合: 講義で扱う監視対象かを再確認し、`メトリクス軽量サービス` 項目を代替参照。
- 指標名が古い表記の場合: コンソール表示名に合わせて補足記録を取り、カリキュラム内で補正メモを加える。
- 文字化けや表示崩れ: `service_metric_catalog.md` を再読み込みし、文字コード設定（UTF-8）で表示し直す。

## 主要リソース
- `service_metric_catalog.md`: 全体のコアカタログ
- 追加コース資産は本ファイル内の表に追記していく。
